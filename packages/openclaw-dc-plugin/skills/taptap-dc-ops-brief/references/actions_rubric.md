# 点赞/回复动作建议 Rubric（简化版）

本文件用于在“只读简报”之外，给出可控的运营动作建议。任何写操作都必须先征得用户许可。

## 点赞（like）建议

优先点赞的评价类型：

- 信息密度高：包含明确问题描述、复现步骤、机型/版本信息
- 具有代表性：反映当前版本最常见的 1-2 个核心问题
- 正向长评：能帮其他用户做决策，且语气友好

不建议点赞的情况：

- 明显引战、辱骂、人身攻击
- 涉及敏感/合规风险

## 官方回复（reply）建议

建议回复的触发条件：

- 负向或中立，且包含可行动点
- 互动较高或对转化影响大
- 同类问题集中出现

不建议直接回复的情况：

- 涉及账号/支付隐私，需要引导走客服渠道
- 需要承诺时间点或赔付

## 执行前确认

调用写操作前，必须向用户展示：

- 目标评价：`review_id` + 内容摘要
- 拟执行动作：点赞/回复
- 拟回复文本（如为回复）
